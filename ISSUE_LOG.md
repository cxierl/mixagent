# MixAgent 问题复盘日志

> 目的：每次改动后记录“问题 -> 根因 -> 修复 -> 防再犯规则”，避免同类问题反复出现。

## 2026-03-22（第 1 轮复盘）

1. 问题：SkillHub 导入经常失败（跨域/CSP/鉴权），导致用户无法安装技能。
- 根因：前端直接 `fetch` 第三方页面，受跨域与页面结构变化影响。
- 修复：内置 SkillHub 推荐技能包（20 条），并在单条/批量导入失败时自动回退到本地推荐包。
- 防再犯：所有外部技能导入流程都必须有“离线可用回退”。

2. 问题：讨论开始前，顶部话题显示旧话题，造成误解。
- 根因：标题直接绑定 `currentTopic_`，未判断是否存在有效讨论上下文。
- 修复：标题显示增加条件（仅在运行中或已有有效消息时显示话题）。
- 防再犯：任何“状态展示”必须区分“配置态”和“运行态”。

3. 问题：未手动创建文件时，右侧出现大量自动生成文件。
- 根因：项目加载时自动创建记录文件（旧方案迁移残留）。
- 修复：改为仅在记录员首次沉淀时懒创建；加载项目时清理历史自动空文件。
- 防再犯：默认不自动生成用户可见文件，除非明确触发写入。

4. 问题：用户担心“优化输入”会直接开始任务。
- 根因：文案不够明确，且优化动作与执行动作在同一操作区。
- 修复：优化完成提示改为“仅改写文本，不会自动开始执行”；运行中禁止优化。
- 防再犯：任何可能被误判为“执行”的按钮，都必须有显式非执行提示。

## 复盘规则（长期）

- 每次功能改动后，必须新增一条日志记录。
- 每条记录必须包含：问题、根因、修复、防再犯规则。
- 若同类问题再次出现，必须补充“为何规则失效”。

## 2026-03-22（第 2 轮复盘）

1. 问题：顶部“项目/团队”控件集中在右上角，右侧拥挤，中间区域空白，操作动线不自然。
- 根因：控件放在 topbar 的 `sp` 之后，被强制右对齐。
- 修复：将项目/团队整组控件移动到导航按钮后，保留右侧仅主题与导出；并压缩输入/下拉宽度。
- 防再犯：顶部工具栏布局遵循“高频配置靠左，状态操作靠右”，避免所有控件堆在一侧。

## 2026-03-22（第 3 轮复盘）

1. 问题：团队历史查询与切换入口不直观，且部分交互细节不完整（查询输入不支持回车、历史弹窗遮罩无法直接关闭）。
- 根因：团队管理流程分散在不同位置，且交互事件绑定不完整。
- 修复：在顶部项目/团队区新增并固定“查询/历史”入口；新增历史团队弹窗列表；查询输入支持回车；历史弹窗支持点击遮罩关闭；“查询定位”改为按团队 ID 定位，避免特殊字符导致异常。
- 防再犯：所有新增入口必须同时补齐“键盘触发 + 遮罩关闭 + 异常字符安全”三项交互校验。

2. 问题：用户担心“新建团队后是否会持久保存”。
- 根因：虽已写入 `localStorage`，但缺少显式可见的“历史可查”链路，用户无法确认保存结果。
- 修复：新增历史团队查询/列表（按更新时间排序），创建或切换后可立即在历史列表中检索与切换。
- 防再犯：涉及持久化的数据能力，必须提供“可见可查”的验证入口，而不只依赖底层存储。

## 2026-03-22（第 4 轮复盘）

1. 问题：页面与模型配置弹窗偏窄，和超宽窗口比例不协调。
- 根因：主布局与弹窗宽度使用固定较小值。
- 修复：调整三栏比例并放宽 `wide` 模态宽度；智能体/技能弹窗也改为宽版。
- 防再犯：所有核心面板必须按 `vw + max-width` 组合做响应式，不使用单一固定宽度。

2. 问题：技能列表缺少分类，且不能按下载热度看优先级。
- 根因：技能模型只有基础字段，未包含分类/下载量/来源元数据。
- 修复：新增技能分类筛选和排序（默认按 SkillHub 下载量）；导入解析支持 category/download/source。
- 防再犯：外部内容导入必须保留可排序元数据，不只保存名称与提示词。

3. 问题：出现“上一位发言未完整结束，下一位就开始”的体验问题。
- 根因：流式返回结束即切换，缺少“完整结束标记”和续写兜底机制。
- 修复：统一要求发言以 `【发言结束】` 结尾；若缺标记自动续写最多两次，再进入下一位。
- 防再犯：多智能体串行发言必须有“完成判据”，不能只依赖网络流结束事件。

4. 问题：右侧自动设定文件数量膨胀（设定1、设定2...），结构不贴合创作场景。
- 根因：按待办数量动态生成编号文件，缺少固定设定维度。
- 修复：改为固定五类设定文件：思维树/世界观/力量/人物/剧情，并按关键词路由记录。
- 防再犯：项目沉淀文件必须先定义稳定信息架构，再做自动化写入。

5. 问题：产出文件面板只能看一行，阅读与回看效率低。
- 根因：文件内容默认折叠且单卡显示信息过少。
- 修复：文件卡默认按内容展开，新增“展开全部/收起全部”，并增强多行预览与换行。
- 防再犯：高频阅读区必须提供批量展开能力和可读性优先的多行展示。

6. 问题：模型编辑窗口易误关闭（闪退感）。
- 根因：模态层通用“点遮罩关闭”策略误覆盖模型编辑场景。
- 修复：模型模态改为仅按钮关闭，不再使用遮罩点击关闭。
- 防再犯：涉及输入中的关键配置窗口默认禁用遮罩关闭，避免误触丢失上下文。

7. 问题：记忆策略入口重复，自动摘要触发不稳定，且无限讨论缺少闭环停机条件。
- 根因：高级设置与记忆面板同时暴露同类策略；自动摘要触发点放在轮内；无限模式仅靠一致性判断。
- 修复：隐藏高级设置里的上下文策略入口，统一在记忆面板调参；自动摘要改为“轮结束后”触发；主持人出现“会议最终闭环宣告”立即停机。
- 防再犯：状态策略只保留单一主入口；轮级动作应在轮结束事件触发；无限循环必须有显式人工终止语义。

## 2026-03-22（第 5 轮复盘）

1. 问题：暂停后点击页面会继续执行，但按钮仍显示暂停态，状态不一致。
- 根因：`checkPause()` 只等待一次；若等待被非预期唤醒且 `paused` 仍为 `true`，流程会误放行到下一发言。
- 修复：`checkPause()` 改为 `while(paused)` 持续阻塞，只有 `paused=false` 才允许继续执行。
- 防再犯：所有“暂停门闩”必须使用状态循环校验，不可只做一次性等待。

## 2026-03-22（第 6 轮复盘）

1. 问题：配置入口分散，学习成本高，缺少“先配模型/技能/智能体/工作流”的一站式路径。
- 根因：功能虽齐全，但交互路径分散在多个按钮和区域，缺少统一入口。
- 修复：新增“工作台配置中心”弹窗，集中展示模型、技能、智能体、工作流状态，并提供一步到位快捷入口。
- 防再犯：复杂系统必须优先提供“单入口配置中心”，再扩展高级功能。

2. 问题：桌面发布流程未闭环，用户无法直接安装体验。
- 根因：虽然已有 Electron 配置，但未完成本机实际打包验证。
- 修复：修复 `package.json` BOM 导致的构建失败，完成 NSIS 打包并产出安装包 `MixAgent Setup 0.1.0.exe`。
- 防再犯：每次发布前必须执行完整打包回归，包含依赖安装、构建和产物核验。

## 2026-03-22（第 7 轮复盘）

1. 问题：用户希望“每个会话是群聊”，并可在会话内快速增减智能体成员。
- 根因：原界面以“智能体列表”为中心，不是“群聊列表”为中心。
- 修复：左侧改为群聊列表（基于项目），支持一键切换；新增“群聊成员管理”弹窗，支持按会话勾选成员。
- 防再犯：后续迭代优先“会话对象”而不是“系统对象”作为主导航入口。

2. 问题：创建项目流程不符合“弹窗创建”的操作习惯。
- 根因：原创建入口是顶部输入框+按钮，交互路径冗长。
- 修复：新增“创建群聊项目”弹窗，包含群聊名、团队名和说明，创建后自动出现在左侧列表。
- 防再犯：高频创建动作必须采用弹窗向导，不依赖分散字段拼装。

3. 问题：模型和技能入口分散，不符合“统一放设置里”的诉求。
- 根因：模型/技能入口分别暴露在导航主区。
- 修复：新增“设置/工作台”统一入口，模型/技能/智能体/工作流由设置中心分发；主区弱化模型/技能导航。
- 防再犯：配置型能力统一归集到设置中心，主界面只保留核心运行路径。

## 2026-03-22（第 8 轮复盘）

1. 问题：切换左侧群聊后，历史对话未恢复，导致“群聊像同一个会话”。
- 根因：项目快照未保存可重建的消息历史。
- 修复：在项目快照中保存精简消息数据（去除 DOM 引用），切换群聊时重绘历史消息。
- 防再犯：所有“会话切换”能力必须包含会话内容层状态，不仅保存配置层状态。

## 2026-03-22（第 9 轮复盘）

1. 问题：用户期望 UI 风格与参考图一致（深色、扁平、群聊中心），原界面视觉仍偏“控制台”。
- 根因：历史样式变量偏蓝色科技风，布局包含过多面板与信息密度。
- 修复：新增 Cherry-like 主题覆盖（深色中性配色、左侧群聊列表、中间主聊天、右侧面板默认隐藏、底部输入区重构）。
- 防再犯：当用户明确给出 UI 参考图时，优先做“整体视觉系统迁移”，而不是局部样式修补。

2. 问题：担心功能改版后出现按钮断链或不可用。
- 根因：大规模调整 HTML 后，`onclick` 链接容易失配。
- 修复：新增一次静态连通性校验（扫描 `onclick` 调用并核对函数定义），确认无断链后再打包。
- 防再犯：每次大改 UI 必须执行“语法校验 + 事件连通性校验 + 打包校验”三步。

## 2026-03-22（第 10 轮复盘）

1. 问题：主界面仍残留旧布局元素（底部策略块、浮动助手、分散入口），与目标参考 UI 不一致。
- 根因：此前只做了样式覆盖，未把旧交互入口彻底收口。
- 修复：主界面改为“左群聊 + 中聊天 + 底部输入”结构；隐藏浮动助手与底部附加设置块；顶部改为参考式导航和窗口按钮区。
- 防再犯：UI 迁移必须同时执行“结构收口 + 样式替换”，不能只做皮肤层覆盖。

2. 问题：模型/技能/智能体在主界面多处可见，配置路径混乱。
- 根因：配置入口散落在导航、输入区和侧栏，缺少统一归档。
- 修复：新增设置中心分区（模型/技能/智能体/工作流），主界面移除模型与技能直接入口，智能体仅在设置中编辑。
- 防再犯：配置型功能必须单入口管理；运行界面只保留任务流与会话流。

3. 问题：群聊未显式加成员时默认带全员，违背“先建群再加智能体”。
- 根因：团队初始化和加载时默认回退到全量智能体。
- 修复：团队默认成员改为空；项目无团队时成员列表也为空；仅在“群聊成员管理”中手动勾选后生效，并在聊天顶端可见可编辑。
- 防再犯：涉及权限/成员绑定的逻辑禁止“隐式全选”，必须显式确认后生效。

4. 问题：顶部窗口按钮仅是视觉占位，无法真实最小化/最大化/关闭。
- 根因：Electron 未开放窗口控制 IPC，渲染层无法调用原生窗口行为。
- 修复：桌面端切换为自绘标题栏（`frame:false`），新增 `mix-win-control` IPC 与 preload 桥接，顶部按钮改为真实窗口控制。
- 防再犯：凡是 UI 中可点击的“系统级按钮”，必须接真实行为或明确不可用状态，不允许长期占位。

## 2026-03-22（第 11 轮复盘）

1. 问题：顶部仍出现“项目胶囊/编辑成员”区，不符合参考图简洁头部。
- 根因：上一版把群聊入口放在 topbar，造成头部信息冗余。
- 修复：移除顶部项目胶囊，仅保留“首页 / 智能体 / +”与窗口按钮，群成员入口下沉到聊天头部。
- 防再犯：topbar 只保留全局导航与窗口控制，不承载会话级信息。

2. 问题：用户希望默认首个会话是单助手对话，而不是普通群聊。
- 根因：默认房间逻辑仍是普通项目（需要智能体成员）。
- 修复：新增固定默认房间“智能助手”（`kind=assistant`），启动时自动进入；该房间仅显示“智能助手”成员并走单助手回复链路。
- 防再犯：默认入口必须严格对齐用户心智模型（首屏即开聊），避免先配队再能说话。

3. 问题：输入区左下角图标过多，操作噪音大。
- 根因：把多个次要动作都塞进了 composer 图标区。
- 修复：左下角收敛为单个“三横线设置”按钮，仅提供“简单模式 / 高级模式”两档发言策略切换。
- 防再犯：输入区仅保留高频动作，低频配置统一收口到设置中心。

4. 问题：设置中心与技能/模型管理仍偏纵向堆叠，阅读和定位成本高。
- 根因：配置区域未采用“左名称列表 + 右配置区”的信息架构。
- 修复：设置中心改为左右分栏；模型/技能弹窗同步改为左右布局，左侧名称列表、右侧配置操作，减少整页滚动。
- 防再犯：配置页默认使用 master-detail（列表-详情）模式，不再堆叠式长页面。

## 2026-03-22（第 12 轮复盘）

1. 问题：左上角仍残留“智能体 / +”入口，破坏用户要求的极简首屏。
- 根因：上一轮只做了结构迁移，没有彻底删除历史导航按钮。
- 修复：移除 topbar 中的 `智能体` 与 `+` 按钮，仅保留 `首页` 与窗口控制；设置入口继续走右上角齿轮。
- 防再犯：当用户明确要求“删除”而不是“隐藏”某入口时，优先直接删 DOM，而不是只做视觉遮挡。

2. 问题：聊天区顶部仍显示标题主行，和“只保留群成员条”的目标冲突。
- 根因：聊天头部保留了标题/轮次/状态的旧层级。
- 修复：Cherry 主题下隐藏 `chat-hd-main`，聊天头只保留成员列表与编辑成员入口，视觉层级更干净。
- 防再犯：会话头部只保留“当前必需信息”，运行状态与轮次不再默认占据首层视觉位置。

3. 问题：群聊列表缺少固定与删除能力，无法形成稳定工作台。
- 根因：项目存储只有创建/加载逻辑，没有列表级管理状态。
- 修复：新增 `pinned` 状态、固定优先排序、群聊卡片悬停操作按钮，以及当前群删除后的安全回退逻辑；默认助手群聊不可删除。
- 防再犯：所有列表型实体都必须具备基础生命周期动作：创建、切换、固定、删除。

4. 问题：群聊卡片和按钮交互反馈偏弱，整体质感不足。
- 根因：之前更多在做布局迁移，缺少成体系的 hover / active / reveal 反馈。
- 修复：补充按钮按压态、群聊卡片悬停抬升、操作按钮悬停显隐、固定态强调色，整体交互更像桌面工作台而不是静态原型。
- 防再犯：每次 UI 改动都必须补齐至少一层“扫描可感知”的交互反馈，而不是只改静态样式。

## 2026-03-22（第 13 轮复盘）

1. 问题：自绘窗口无法拖动，桌面版像“卡死的网页”。
- 根因：Electron 使用无边框窗口，但页面顶栏没有声明系统拖拽区。
- 修复：将 topbar 设为 `-webkit-app-region: drag`，并对窗口按钮/输入控件显式设为 `no-drag`，恢复窗口拖动与按钮点击。
- 防再犯：凡是 frameless 桌面壳，必须同时交付“drag 区 + no-drag 交互区”。

2. 问题：底部三横线菜单点击无反应。
- 根因：全局 `document.click` 在同一点击事件中把下拉层立即关闭，导致菜单一闪即灭。
